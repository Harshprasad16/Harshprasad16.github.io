AI / Computer Vision
Neuroclass.AI

An AI-powered classroom analytics platform that monitors and analyzes student engagement in real time using computer vision — tracking attention levels, participation patterns, and classroom interactions.

Python Computer Vision AI/ML Analytics Real-Time Processing
Problem Statement

Teachers in large classrooms struggle to gauge student engagement manually. Passive or distracted students often go unnoticed until it's too late — affecting academic outcomes. There was no affordable, automated system to quantify classroom attention in real time.

My Approach
  • Built a computer vision pipeline using Python to detect faces and track gaze direction from a webcam feed.
  • Designed attention scoring logic based on head pose estimation and eye tracking metrics.
  • Created a dashboard layer to visualize engagement trends per student over a session.
  • Optimized inference speed to run in near real-time on standard hardware.
Challenges Faced
  • Handling varying lighting conditions in classroom environments which significantly impacted face detection accuracy.
  • Reducing false positives in attention scoring when students were looking at notes vs looking away from the screen.
  • Balancing real-time processing speed with model accuracy — had to optimize the pipeline to avoid frame drops.
  • Designing a meaningful "engagement score" metric that was statistically reliable and not misleading to educators.
Tech Stack
Python OpenCV MediaPipe NumPy Matplotlib

🎯 Attention scoring system implemented

🏫 Applicable to any classroom setup

IoT / Hardware
SafeX Riding

A hardware-based road safety system where a motorcycle starts only when the rider wears a helmet. Led a 6-member team to design and implement a sensor-based safety mechanism.

IoT Sensors Hardware Team Lead Embedded Systems
Problem Statement

Helmet non-compliance is one of the leading causes of fatal motorcycle accidents in India. Despite laws mandating helmet use, enforcement is inconsistent. The goal was to build a system that makes it physically impossible to ride without wearing one — enforced at the ignition level.

My Approach
  • Designed a pressure/contact sensor circuit embedded inside a helmet to detect when it is properly worn.
  • Integrated the sensor signal with a microcontroller (Arduino) that controls the motorcycle's ignition relay.
  • Engineered failsafe logic to prevent false positives (e.g., helmet placed on seat but not worn).
  • Led a cross-functional 6-member team covering hardware wiring, embedded coding, and prototype testing.
Challenges Faced
  • Distinguishing between "helmet worn" and "helmet held" — required combining pressure and strap-clip detection logic.
  • Ensuring the ignition relay circuit was safe and did not interfere with existing motorcycle electronics.
  • Waterproofing the sensor assembly to make it viable for all weather conditions.
  • Coordinating a 6-person team with mixed skill levels across hardware and software components within a tight project timeline.
Tech Stack
Arduino C (Embedded) Relay Module Pressure Sensors Circuit Design
Outcomes & Impact
🛡️ Working prototype built & tested
👥 Led 6-member team end-to-end
🏍️ Ignition-level enforcement achieved
🇮🇳 Addresses a real Indian road safety gap

Mobility / Accessibility Design
Fasten

An innovative mobility assistance solution designed for elderly and physically challenged individuals. Focused on user safety, accessibility, and ease of movement through practical design concepts.

Accessibility Research Design Project Lead User-Centered Design
Problem Statement

Elderly and physically challenged individuals often struggle with existing mobility aids that are either too complex, too heavy, or not designed with Indian usage contexts in mind. Falls and mobility-related injuries significantly affect quality of life. There was a clear need for a practical, safe, and user-friendly mobility assistance solution.

My Approach
  • Conducted research into the daily mobility challenges of elderly and physically challenged users through observation and interviews.
  • Analyzed existing solutions (walkers, crutches, motorized scooters) and identified key gaps in usability and safety.
  • Designed a modular mobility aid concept with focus on stability, lightweight build, and adaptability to different environments.
  • Led the project from ideation through prototype design, incorporating feedback loops at each stage.
Challenges Faced
  • Balancing structural stability with the need for a lightweight and portable design — most existing aids sacrifice one for the other.
  • Designing for diverse physical conditions (partial paralysis, arthritis, post-surgery recovery) within a single adaptable system.
  • Incorporating safety features (anti-tip mechanism, grip locks) without adding complexity to the user interaction.
  • Working within resource and budget constraints typical of a student design project while maintaining functional fidelity.

User Research Ergonomics Prototype Design Safety Engineering Accessibility Standards
